# presence-manager

ラボメンバーの在席状況をリアルタイムで共有・管理するWebアプリです。スマートフォン・PCどちらからでもワンタップでステータスを更新できます。

## 概要

研究室の在席状況をリアルタイムで可視化するシステムです。カードをタップするだけでステータスを変更でき、管理者は在室時間の週次集計も確認できます。Vercel上にデプロイし、研究室内のどの端末からもアクセス可能です。

## 主な機能

- **リアルタイム在席ボード** : メンバーごとのステータスをカード形式で表示
- **ワンタップ更新** : カードをタップしてポップアップからステータスを選択
- **週次在室時間の集計** : バックエンドで在室時間を自動集計し、カードに表示
- **夜間自動省電力モード** : 22時〜翌8時は更新間隔を延長してサーバー負荷を軽減
- **管理画面** : メンバー・ステータス一覧の追加・変更が可能
- **楽観的UI更新** : サーバー応答を待たず即座に画面を更新し、体感速度を向上

## 技術スタック

| 領域 | 技術 |
|------|------|
| フロントエンド | HTML / CSS / Vanilla JavaScript |
| バックエンド | Python (Vercel Serverless Functions) |
| データベース | PostgreSQL |
| インフラ・デプロイ | Vercel |

## アーキテクチャ

```
index.html (フロントエンド)
    │
    ├── GET  /api/members   → メンバー一覧・ステータス取得
    ├── GET  /api/settings  → ステータス一覧・表示設定取得
    └── POST /api/update    → ステータス更新
          │
          └── PostgreSQL (在席状態・週次時間の永続化)
```

フロントエンドはSPAとして動作し、APIから取得したデータをもとに動的にカードを描画します。楽観的UIパターンを採用し、APIへのリクエストと並行して画面を即時更新します。

## 工夫した点

- **タイムゾーン対応** : Vercelサーバーの時刻とJSTのズレを吸収するためUTC変換処理を実装
- **夜間省電力モード** : `getHours()`で時間帯を判定し、更新インターバルを動的に切り替え
- **ステータスカラーの自動生成** : 未定義のステータスに対してハッシュ関数でHSLカラーを自動割り当て
- **モバイルUX** : `-webkit-tap-highlight-color`除去やタップフィードバックアニメーションでネイティブアプリに近い操作感を実現

## ローカルでの実行

```bash
# 依存関係のインストール
pip install -r reqirements.txt

# Vercel CLIで開発サーバーを起動
vercel dev
```

環境変数として `DATABASE_URL`（PostgreSQL接続文字列）を設定してください。
